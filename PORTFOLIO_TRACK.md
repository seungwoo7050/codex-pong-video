# 포트폴리오 트랙 (Vrew 웹 애플리케이션 개발자 맞춤)

목표: “웹에서 리플레이를 재생하고, FFmpeg 기반 비동기 파이프라인으로 MP4를 생성하며, 검증/실패모드/진행률까지 갖춘 시스템”을 만든다.

---

## 1) 이번 트랙에서 반드시 만드는 것

### v0.5.0 (MVP)
- 리플레이 기록/저장/목록/뷰어
- Export MP4 (잡 기반, 워커 분리, 진행률)
- “의미없는 영상더미” 방지 검증(필수)
  - ffprobe 구조 검증
  - 두 시점 프레임 비교(같으면 실패)

### v0.6.0 (품질 증명)
- WebGL 렌더러 + Canvas2D fallback
- WebAudio 1개 이상 실제 사용
- reflow/perf 감사 리포트(2화면 이상)
- UTF-8 회귀 테스트(한글/이모지, REST/WS/DB)

---

## 2) 3분 데모 시나리오(면접용)

1) 경기 종료 → 리플레이 생성됨  
2) 리플레이 뷰어(play/seek/speed)  
3) Export MP4 클릭  
4) 진행률 실시간 표시  
5) MP4 다운로드 후 재생(“진짜 영상”)  

---

## 3) 공고 기술과의 매핑(면접에서 말할 포인트)

- React/TS: 뷰어 + 진행률 UI
- Node/TS: 워커(큐 소비 + 프로세스 I/O)
- FFmpeg: export + progress 파싱 + 검증(성공 조건을 엄격히 정의)
- WebGL: GPU 경로 + fallback 설계
- WebAudio: 최소 1개 오디오 큐
- IPC/async I/O: Redis Streams + job 상태 머신 + WS 푸시
- UTF-8: 한글/이모지 회귀 테스트로 증명

---

## 4) 이번 트랙에서 버리는 것(범위 컷)

- 친구/채팅/토너먼트/관전/운영툴/모니터링은 제외
- 기능 확장보다 “작동 + 실패모드 + 검증 + 근거” 우선
