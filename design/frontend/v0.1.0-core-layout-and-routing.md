# v0.1.0 프런트엔드 설계: 코어 레이아웃 및 라우팅

## 1. 목적 / 범위
- React + TypeScript SPA의 기본 라우팅과 스타일 프레임을 마련한다.
- 랜딩, 로비, 게임(실시간 테스트) 화면을 최소 구현한다.
- REST 및 WebSocket 연결 주소를 환경변수 기반으로 분리한다.

## 2. 화면 / 동작
- 랜딩 페이지(`/`)
  - 서비스 소개 텍스트와 로비/게임 이동 버튼 제공.
- 로비 페이지(`/lobby`)
  - 입력 폼: 플레이어 A/B 이름, 점수.
  - `POST /api/games` 호출로 테스트 경기 생성.
  - `GET /api/games` 목록을 렌더링.
- 게임 페이지(`/game`)
  - WebSocket 연결 상태 표시.
  - 입력값을 `/ws/echo`로 전송하고 수신 로그를 표시.

## 3. 구조 / 폴더
- `src/pages`: 각 화면 컴포넌트 (Landing, Lobby, Game).
- `src/hooks/useEchoSocket.ts`: WebSocket 연결/메시지 상태 관리 훅.
- `src/constants.ts`: API/WS 기본 URL 정의 (환경변수 fallback).
- `App.tsx`: 헤더 네비게이션 + React Router 라우팅.
- 스타일: `index.css`에서 전역 색상/레이아웃 정의.

## 4. API / 통신
- REST: `API_BASE_URL` (기본 `http://localhost:8080`).
- WebSocket: `WS_BASE_URL` (기본 `ws://localhost:8080`).
- Docker Compose 시 Nginx가 80 포트에서 두 서비스로 프록시.

## 5. 테스트 전략
- v0.1.0에서는 수동 확인(REST/WS 응답)과 빌드 성공을 목표로 한다.
- 이후 버전에서 React Testing Library 기반 스냅샷/행동 테스트 추가 예정.

## 6. 마이그레이션 / 향후 작업
- v0.2.0 이후 인증/프로필 UI 추가 시 라우팅 가드 적용.
- 게임 화면을 실제 렌더링/캔버스 형태로 확장 예정.
