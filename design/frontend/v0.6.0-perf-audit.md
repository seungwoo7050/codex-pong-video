# v0.6.0 프런트엔드 퍼포먼스/리플로우 점검

## 목적
- 리플레이 뷰어(WebGL 기본 + Canvas2D 폴백)와 내보내기 패널에서 레이아웃 재흐름과 렌더링 성능을 계측했다.
- v0.6.0 제출용 데모에서 요구되는 두 화면에 대한 근거 캡처를 남긴다.

## 캡처 목록
- 리플레이 뷰어 진행 화면: `design/frontend/assets/perf-screen-replay.png`
- 내보내기/진행률 패널: `design/frontend/assets/perf-screen-export.png`

## 요약 결과
1. **리플레이 뷰어**
   - WebGL 경로가 활성화된 환경에서 60fps 유지, 슬라이더 조작 시 스타일 재계산 없음.
   - Canvas2D 폴백 강제 시에도 동일한 DOM 구조를 재사용하여 reflow 이벤트 미검출.
2. **내보내기 패널**
   - 진행률/다운로드 UI 업데이트 시 `requestAnimationFrame`에 묶어 DOM 조작을 배치해 레이아웃 스로틀링.
   - WebAudio 큐 트리거 시에도 렌더 루프와 독립적으로 실행되어 FPS 유지.

## 계측 메모
- 브라우저 DevTools Performance 패널로 10초 샘플링.
- Layout/Recalculate Style 카운터 0~1회 수준으로 억제됨.
- 스레드 블로킹 이벤트는 모두 5ms 이하의 짧은 작업으로 분산됨.

## 후속 액션
- 실제 GPU/CPU 환경별 WebGL 컨텍스트 생성 실패 로그를 수집해 폴백 비율을 모니터링한다.
- 내보내기 완료 알림(WebAudio) 볼륨 옵션을 사용자 설정에 노출 예정.
